# ワークフロー アクション クイック リファレンス (SharePoint 2013 ワークフロー プラットフォーム)
このリファレンスでは、SharePoint Designer 2013の現在のビルドでサポートされているワークフロー アクション (利用可能でないものも含む) を示します。
## SharePoint Designer 2013のワークフロー アクション
<a name="bkm_WorkflowActions"> </a>

以下に、SharePoint 2013 ワークフロー プラットフォームで使用できるワークフロー アクションに関するリファレンスを示します。SharePoint Designer 2013 では、SharePoint 2013 ワークフロー プラットフォームに加え、SharePoint 2010 ワークフロー プラットフォームもサポートしています。2010 のプラットフォーム用のワークフロー アクションについては、「 [ワークフロー アクション クイック リファレンス (SharePoint 2010 ワークフロー プラットフォーム)](workflow-actions-quick-reference-sharepoint-2010-workflow-platform.md)」を参照してください。
  
    
    

### コア アクション
<a name="bkm_CoreActions"> </a>

コア アクションは、特によく実行されるアクションであり、容易にアクセスできるようにまとめてグループ化されています。
  
    
    

**表 1. コア アクションのリファレンス**


|**アクション**|**説明**|
|:-----|:-----|
|コメントを追加する  <br/> |参考のために有益なコメントをワークフロー デザイナーに残しておくことができます。これが特に役立つのは、ワークフローで共同作業を行っている他のユーザーがいる場合です。  <br/> |
|日付に時刻を追加する  <br/> |日付に特定の時間 (分、時間、日、または月単位で指定。年単位はサポートされていません) を加え、出力値を変数として格納します。日付には、現在の日付、特定の日付、またはルックアップを指定できます。'現在の日付' の値は、協定世界時の深夜 0 時を返します。  <br/> |
|辞書を作成  <br/> |キーと値のペアから成る Dictionary 変数を作成します。  <br/> > **メモ**> Dictionary では JSON 表記を使用してデータを保存します。           Dictionary 変数の詳細については、「 [SharePoint Designer 2013 の辞書アクションを理解する](understanding-dictionary-actions-in-sharepoint-designer-2013.md)」を参照してください。  <br/> |
|HTTP Web サービスを呼び出す  <br/> |HTTP Web サービスに対するメソッド呼び出しとして機能し、JSON 形式を使用してデータを返します。基本認証は RequestHeader によってサポートされています。  <br/> Dictionary 変数の詳細については、「 [SharePoint Designer 2013 の辞書アクションを理解する](understanding-dictionary-actions-in-sharepoint-designer-2013.md)」を参照してください。  <br/> |
|辞書内のアイテムをカウントする  <br/> |指定した辞書内のアイテムの数を返します。  <br/> |
|計算を行う  <br/> |算術演算を実行し、結果の値を変数に格納します。  <br/> > **メモ**> SharePoint 2013では、このアクションは **Double** 数値型のみをサポートします。整数はサポートされません。文字列での "+" 演算子 (文字列連結演算子) の使用はサポートされていません。          |
|辞書からアイテムを取得する  <br/> |辞書変数から特定のアイテムを返します。  <br/> |
|履歴リストに記録する  <br/> |定義済みのメッセージ アイテムのリストからワークフローの履歴リストにメッセージを書き出します。  <br/> |
|指定した期間だけ停止する  <br/> |指定した時間 (日、時間、または分単位) だけ、ワークフローの実行を停止します。  <br/> |
|指定した日まで停止する  <br/> |指定した日時までワークフローの実行を停止します。  <br/> |
|電子メールを送信する  <br/> |指定したワークフロー イベントが発生したときに、事前定義済みのメッセージを含んだ電子メール メッセージをユーザーまたはグループに自動送信します。  <br/> > **重要**> サイトが信頼済みサイトの一覧に追加されていない場合、電子メールは Outlook の [迷惑メール] フォルダーに振り分けられます。           |
|日付/時刻フィールドの時刻部分を設定する  <br/> |タイムスタンプを作成し、出力値を変数に格納します。時刻は時間および分の各単位で設定でき、現在の日付、特定の日付、またはルックアップを追加できます。  <br/> |
|ワークフローの状態を設定する  <br/> |ワークフローの状態を設定します。  <br/> |
|ワークフローの変数を設定する  <br/> |ワークフロー変数に値を設定します。また、このアクションは、ワークフローによってデータを変数に割り当てる場合にも使用できます。  <br/> |
|次のステップに進む  <br/> |フローで制御される次のステップを指定します。  <br/> |
   

### 調整アクション
<a name="bkm_CoreActions"> </a>

調整アクションは、SharePoint 2010 ワークフロー プラットフォームに基づいたワークフローを呼び出すために使用されます。調整アクションの詳細については、「 [SharePoint Designer 2013 での連携アクションを理解する](understanding-coordination-actions-in-sharepoint-designer-2013.md)」を参照してください。
  
    
    

**表 2. 調整アクションのリファレンス**


|**アクション**|**説明**|
|:-----|:-----|
|リストのワークフローを開始する  <br/> |SharePoint 2010 ワークフロー プラットフォームに基づいたリスト ワークフローを開始します。  <br/> > **メモ**>  [リストのワークフローを開始する] には次の問題があります。>  2010 のワークフローで '課題' タイプのフィールドに TaskProcess アクションがある場合、そのフィールドはパラメーターとして使用できません。>  同じ 2010 のワークフローに対して複数の呼び出しが行われた場合、2013 のワークフロー参照機能では結果が複数のデータ ソースになります。これらのデータ ソースはすべて同じものです。>  2013 の変数名には、'?' や '#' のような特殊文字を含めることができません。2010 のワークフローに特殊文字が含まれている場合、2013 のワークフローではそれらの文字が 16 進数コードに変換されます。          |
|サイトのワークフローを開始する  <br/> |SharePoint 2010 ワークフロー プラットフォームに基づいたサイト ワークフローを開始します。  <br/> > **メモ**>  [リストのワークフローを開始する] には次の問題があります。>  2010 のワークフローで '課題' タイプのフィールドに TaskProcess アクションがある場合、そのフィールドはパラメーターとして使用できません。>  同じ 2010 のワークフローに対して複数の呼び出しが行われた場合、2013 のワークフロー参照機能では結果が複数のデータ ソースになります。これらのデータ ソースはすべて同じものです。>  2013 の変数名には、'?' や '#' のような特殊文字を含めることができません。2010 のワークフローに特殊文字が含まれている場合、2013 のワークフローではそれらの文字が 16 進数コードに変換されます。          |
   

### リスト アクション
<a name="bkm_ListActions"> </a>

リスト アクションには、リストおよびリスト アイテムの操作に使用されるアクションがまとめられています。
  
    
    

**表 3. リスト アクションのリファレンス**


|**アクション**|**説明**|
|:-----|:-----|
|アイテムをチェックインする  <br/> |チェックアウトされていたアイテムをチェックインします。アイテムのチェックインは、ドキュメント ライブラリからのみ行えます。  <br/> > **注意**> チェックアウトしていないアイテムをチェックインしようとすると、ワークフローがクラッシュします。           |
|アイテムをチェックアウトする  <br/> |アイテムをチェックアウトします。ドキュメントのチェックアウト前に、アイテムがチェックインされているかどうかがワークフローによって検証されます。アイテムのチェックアウトは、サイト内のライブラリからのみ行えます。  <br/> > **注意**> チェックインしていないアイテムをチェックアウトしようとすると、ワークフローがクラッシュします。           |
|ドキュメントをコピーする  <br/> |現在のリストにあるドキュメントを異なるドキュメント ライブラリ リストにコピーします。  <br/> |
|リスト アイテムを作成する  <br/> |指定したリストに新しいリスト アイテムを作成します。新しいアイテムにはフィールドや値を入力できます。このアクションは、特定の情報を使用して新しいアイテムを作成する場合にいつでも使用できます。  <br/> |
|アイテムを削除する  <br/> |アイテムを削除します。  <br/> > **メモ**> このアクションは、ワークフロー マネージャー ワークフロー エンジンを実行しているコンピューターでは終了され、 **System.InvalidOperationException** 例外をスローします。回避策はありません。          |
|アイテムのチェックアウトを破棄する  <br/> |変更内容を破棄し、アイテムがチェックアウトされて変更が加えられている場合はチェックイン状態に戻します。  <br/> > **注意**> チェックアウトされていないアイテムのチェックインを試みると、ワークフローがクラッシュします。           |
|現在のアイテムにフィールドを設定する  <br/> |現在のアイテムの指定したフィールドに指定した値を設定します。  <br/> > **メモ**> フィールドの値が変更されるまでワークフローを停止する場合は、このアクションの代わりに、" **リスト アイテムでイベントが発生するまで待つ** " アクションを使用します。          |
|ドキュメントを翻訳する  <br/> |ドキュメントを特定の言語に翻訳する  <br/> > **メモ**> あらかじめ構成された機械翻訳サービス アプリケーションが必要です。           |
|リスト アイテムを更新する  <br/> |リスト アイテムを更新します。フィールドを指定し、それらのフィールドに新しい値を設定できます。  <br/> |
|リスト アイテムでイベントが発生するまで待つ  <br/> |[強化された Office 2010 のアクション] 指定したリスト アイテムのイベントが発生するまで、ワークフローの現在のインスタンスを停止します。このアクションは、 **ItemUpdated** と **ItemAdded** の 2 つのイベントの発生をリッスンします。 <br/> |
|現在のアイテムでのフィールドの変更を待つ  <br/> |現在のアイテム上のフィールドが特定の値に等しくなるまで待ちます。  <br/> |
   

### プロジェクト アクション
<a name="bkm_ProjectActions"> </a>

プロジェクト アクションは、Microsoft Project の統合をサポートします。Project ベースのワークフローに使用します。プロジェクト アクションはすべて SharePoint Designer 2013で新しく追加されました。
  
    
    

**表 4. プロジェクト アクションのリファレンス**


|**アクション**|**説明**|
|:-----|:-----|
|現在のアイテムからプロジェクトを作成する  <br/> |現在のアイテムを取得し、SharePoint ファームの PWA サイトに新しいプロジェクトを作成します。  <br/> |
|プロジェクト フィールドを設定する  <br/> |Project Server 上の特定のフィールドに値を設定します。  <br/> > **メモ**> このアクションを実行するには、まずプロジェクトをチェックインしておく必要があります。プロジェクトがチェックインされていない場合、ワークフローは終了し、ユーザーはこのプロジェクトを Project Web App で開くことができません。           |
|プロジェクト ステージの状態を設定する  <br/> |プロジェクト ステージの状態を設定します。  <br/> > **メモ**> 現在のプロジェクトがチェックアウトされている場合は、例外が発生します。           |
|概念リストのステータス フィールドを設定する  <br/> |現在のプロジェクトに関連付けられている元のリスト アイテムの状態を更新します。  <br/> |
|プロジェクト イベントを待機する  <br/> |特定のプロジェクト イベントが発生するまで待機します。  <br/> |
   

### タスク アクション
<a name="bkm_TaskActions"> </a>

タスク アクションでは、SharePoint 2010 ワークフロー プラットフォームに基づいたワークフロー内から、SharePoint 2013 ワークフロー プラットフォームに基づいたワークフローを呼び出すことができます。
  
    
    

**表 5. タスク アクションのリファレンス**


|**アクション**|**説明**|
|:-----|:-----|
|タスクを割り当てる  <br/> |ワークフロー タスクをユーザーに割り当て、タスク完了の期限を設定します。  <br/> |
|タスク プロセスを開始する  <br/> |複数のユーザーに対してタスクを作成し、カスタマイズされたプロセスからタスクを取得できるようにします。  <br/> |
   

### ユーティリティ アクション
<a name="bkm_UtilityActions"> </a>

ユーティリティ アクションは、文字列を操作したり、日付の間隔を検出したりするアクションです。
  
    
    

**表 6. ユーティリティ アクションのリファレンス**


|**アクション**|**説明**|
|:-----|:-----|
|文字列の末尾からサブ文字列を抽出する  <br/> |文字列の末尾から指定した数の文字をコピーし、結果の値を変数に格納します。  <br/> |
|文字列のインデックスに基づいてサブ文字列を抽出する  <br/> |文字列内で、指定したインデックスから末尾までの文字をコピーし、結果の値を変数に格納します。  <br/> > **メモ**> Microsoft SharePoint Designer 2013 のインデックス値は 0 から始まりますが、SharePoint Designer 2010 の値は 1 からインデックスが作成されるので注意が必要です。           |
|文字列の先頭からサブ文字列を抽出する  <br/> |文字列の先頭から指定した数の文字をコピーし、出力値を変数に格納します。  <br/> |
|文字数を基にインデックスから文字列のサブ文字列を抽出する  <br/> |文字列中の指定の開始インデックスから指定の文字数分のサブ文字列をコピーし、その値を変数に格納します。  <br/> > **メモ**> Microsoft SharePoint Designer 2013 のインデックス値は 0 から始まりますが、SharePoint Designer 2010 の値は 1 からインデックスが作成されるので注意が必要です。           |
|日付間の間隔を求める  <br/> |2 つの日付の時間の差 (分、時間、または日単位) を計算し、結果の値を変数に格納します。  <br/> |
|文字列をトリミングする  <br/> |文字列の先頭と末尾から空白を削除します。  <br/> |
|文字列からサブ文字列を検索する  <br/> |文字列から特定のサブ文字列を探し、サブ文字列の開始位置を示すインデックスを返します。  <br/> |
|文字列内のサブ文字列を置換する  <br/> |特定のサブ文字列を別のサブ文字列で置き換えます。  <br/> |
|文字列をトリミングする  <br/> |文字列の先頭と末尾から空白を削除します。  <br/> |
