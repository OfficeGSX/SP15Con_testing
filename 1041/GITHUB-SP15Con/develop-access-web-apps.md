# Access Web アプリの開発
Microsoft Access 2013 を使用して、社内やクラウドで共同作業用の Web ベースのアプリケーションを簡単に作成、展開、および管理する方法を説明します。
## 概要
<a name="dk2_DevelopingAccess15WebApps_Introduction"> </a>

Microsoft Access 2013 は、以前の Windows 開発で行われたのと同じ方法で Web 開発を簡素化するように設計されています。Access 2013 により、対象分野の専門家は、業務を遂行するために使用できるアプリケーションを迅速に作成することができます。
  
    
    

  
    
    

## Access 2013 の新機能
<a name="dk2_DevelopingAccess15WebApps_whatsNewInAccess15"> </a>

Access 2013 の特徴の 1 つは、ユーザーがデータ中心の Web アプリケーションを作成できるように設計された新しいアプリケーション モデルです。プログラミングの経験があまりないユーザー、またはまったく経験のないユーザーは、ビジネス ニーズに対応するアプリケーションを急いで作成して、同僚、友人、家族などと共有しなければならないことがよくありますが、Access 2013 は、選択した条件を基にテーブルやフォームを作成することで、ユーザーがすぐに使用できる対話式の方法を提供します。
  
    
    
Access 2013 は、データの格納用に SQL Server を使用しながら、Microsoft SharePoint 2013 を使用してアプリケーションのフロントエンドをホストします。Access 2013 は、Office 365 サービスや SQL Azure サービスを利用して、Access アプリケーションをインターネットに展開する手段を提供します。Access 2013 アプリの共有は、そのアプリの URL を目的のユーザーに送信するくらい簡単です。
  
    
    
Access 2013 アプリの新機能の詳細については、「 [Access の新機能](what-s-new-in-access.md)」を参照してください。
  
    
    

## Access 2013 アプリケーションの作成
<a name="dk2_DevelopingAccess15WebApps_CreatingAnAccess15App"> </a>

多くの SharePoint Server 2013 アプリケーション サービスとは異なり、Access Services 2013 は、Visual Studio での Access アプリ開発に使用できる API を公開しません。Access 2013 は、Access 2013 アプリの開発に使用する環境です。
  
    
    
Access 2013 アプリの開発方法の詳細については、「 [Access 2013 で Web アプリの作成とカスタマイズを行う方法](http://msdn.microsoft.com/library/628745f4-82e9-4838-9726-6f3e506a654f%28Office.15%29.aspx)」を参照してください。
